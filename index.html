Upcoming Features

  • I plan to add support for the Ore Dictionary.
  • Missing ingredients shall be auto-crafted, if their recipes are set up on the machine as well.
  • Recipe chip changes
    • I want the chips to have an greater role on the mod. I will allow portable crafting through them (as simple as with the crafter machine).
    • I want to remove the encoder. The idea is to move that feature to the chips themselves. That is: blank chips are encoded by placing the ingredients on a grid (new GUI). So, the encoding happens only client-side.
    • New GUI for the chips. It has the output slot to automatically consume the ingredients (like the crafter does). Also, it has a crafting grid.
      • Blank chips will have the grid interactive (semi-regular).
      • Encoded chips will just render the ghost items. Missing ingredients (on the player's inv) will appear red.
  • Portable crafting grid. Should I add this? Hmm, probably I should append a regular crafting table to the portable xact.
  • Portable xact (crafter) device. It should mimic the XACT crafter machine, but is portable. Differs at:
    • Power/energy requirement?
    • Is portable
    • Has different amount of slots (less, or more?)

Also, when shift-clicking on the XACT machine, it transfers the contents (perhaps) ?

  • TESR and a model.
    • Still don't know how I'm gonna mode that.
  • Automation modules.
    • I plan to support ISpecialInventory (for BC-related stuff).
    • Some sort of ISidedInventory implementation for all other mods.
  • IC2: UU-matter recipes (make a machine named XACT Matter Transformer). It might have all the recipes already stored. Or use a new item for transforming UU-matter into stuff.
